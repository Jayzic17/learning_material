
go mod init [project_name]
    * initializes the current working directory as a Go application (Module)
        * ideally the folder name should be the same as the project name
    * generates a go.mod file: defines the Go application (Module), any dependencies, and the version of Go being used
go mod tidy
    * cleans up the go.mod and go.sum files by adding missing dependencies, removes unused ones, and updates the go.sum file, which is just a checksum
go mod vendor
    * generates (if it doesn't already exist) the vendor directory, which contains a local copy of all the third-party package source code your project depends on, as specified in your go.mod file
go mod graph
    * returns the dependency tree of your Go application
go run main.go
    * runs your main.go
go run .
    * runs the whole directory - you would run this if you have multiple entry points (i.e multiple main() functions)
go clean
    * cleans up disk space by removing cache and build artifacts in your Go application (Module) and reformates your files like '$ terraform fmt'
go clean -modcache
    * cleans the cache of your Go application (a.k.a the GOMODCACHE environment variable)
grep -R "[dependency_name]" $(go env GOMODCACHE)
    * recursivly checks the cache of your Go application (a.k.a the GOMODCACHE environment variable) for the given dependency
    * good for if you're looking for indirect dependencies that get used as part of the main dependencies you specify in your go.mod file

--ENVIRONMENT VARIABLES---
GOPRIVATE: comma-deliniated list of modules you make sure Go knows are private, meaning these modules aren't accessible from the public internet
